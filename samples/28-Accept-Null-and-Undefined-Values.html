28 - Accept Null and Undefined Values

When using the 'UNDEF-OK' option with TypeCzech creation, 'null' and 'undefined' values are allowed to pass through error free in parameter type checking. The values 'null' and 'undefined' are still however considered empty & valueless. In other words, null and undefined are treated as type wildcards.
  • null - absence of any value
  • undefined - an un-assigned value
The 'UNDEF-OK' option is only useful with type shapes, and thus these functions:
  • valid(actual_arguments, expected_shape)
  • validExtras(actual_arguments, expected_shape)
  • validUnion(actual_arguments, possible_shapes)
  • validUnionExtras(actual_arguments, possible_shapes)

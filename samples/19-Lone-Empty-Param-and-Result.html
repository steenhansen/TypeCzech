19 - Lone Empty Parameter and Result

Use type_czech.valueless(arguments, expected_empty) to verify that a lone parameter or return result is not empty or valueless. Good for ensuring that a function's single string, array, and object parameter and or result has a useful value. When a parameter corresponds to 'EMPTY-ERROR' and its contents are valueless then an error occurs.
Empty or
Valueless Data
  • null
  • undefined
  • '' - empty string
  • NaN - not a number
  • [ ] - no elements in array
  • { } - no members in object
  • new Date('')
  • new RegExp('')
  • Empty Categories
  • 'EMPTY-OK'
    Valueless is ok
  • 'EMPTY-ER'
    Valueless is an Error
  • 'EMPTY-IGNORE'   functions, symbols,
    • and booleans cannot be empty 



The odd error message for the empty array, [ ], stems from the fact that the function arguments object is array-like. This is a bad error message edge case.

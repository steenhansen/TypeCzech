26 - Type and Empty Abbreviations

Abbreviations can be used to make type and empty assertions shorter.
  • Type Shortcuts
    • B - Boolean
    • R - RegExp
    • D - Date
    • S - String
    • F - Function
    • Y - sYmbol
    • N - Number
    • I - BigInt
    • A - Array
    • O - Object
  • Empty Shortcuts
    • 'EM-OK'
      'EMPTY-OK'
    • 'EM-ER'
      'EMPTY-ERROR'
    • 'EM-IG'
      'EMPTY-IGNORE'

    The Pre_Person() function first verifies that an object is the construction parameter, then that object is tested for the correct keys and types, finally all keys are checked for being valueless.

    After which the POST_Person() function first certifies that the constructor returns an array result, then the two elements are established to be the correctly ordered types, lastly the last name is checked for not being empty. A valueless birth date returned is allowed.

    The errors:
    • Queen Boudica does not have a valid birth date for PRE check, but this is OK on POST check
    • Madonna has no last name which causes two error messages in both PRE and POST

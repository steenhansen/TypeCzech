15 - Array and Object Types with Parameter and Results

Typeless arrays can be checked for without giving any dimensionality, if [ ]s are used then an array must define at least one type.
  • Valid Type: 'Array'
  • Not Valid Type : '[ ]'
  • Valid Type : [ 'String' ]
And, 'Object' can also be a type without specifying any properties. But if { }s are used then an object must define at least one key and value.
  • Valid Type : 'Object'
  • Not Valid Type : '{ }'
  • Valid Type : { a_key: 'String' }
Alan Turing is used to show the error messages that occur when an object is validated against the 'array' type in the PRE_Person() checking function. The reverse, testing an array against the 'object' type gives a lengthy error message from the POST_Person() function.

11 - Class Parameters, Results, and Methods

Classes with multiple PRE and POST checking functions must be wrapped up in objects like below for everything to be linked up. 
function PRE_check_yourClass(init_const){ // before constructor }

function POST_check_yourClass(result_object){ // after constructor }

function PRE_check_yourMethod(method_input){ // before method }

function POST_check_yourMethod(method_output){ // after method }

yourClass_PRE_checks = {yourClass:PRE_check_yourClass,
                        yourMethod:PRE_check_yourMethod}

yourClass_POST_checks = {yourClass:POST_check_yourClass,
                         yourMethod:POST_check_yourMethod}

class yourClass {
  constructor(init_const){ }
  yourMethod(method_input){ 
    return method_output
  }
}

yourClass = type_czech.link(yourClass, 
                             yourClass_PRE_checks,
                             yourClass_POST_checks)

Note that the type_czech.link() call must come after the class declaration, otherwise you get the below ReferenceError. 
can't access lexical declaration 'yourClass' before initialization

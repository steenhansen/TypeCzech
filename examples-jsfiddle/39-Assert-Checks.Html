39 - Check Asserts

When working with promise based functions we don't have full control of the lifecycle. So we can instead use an assert construct instead to check variables.

Code containing an check_assert() must ensure that TypeCzech is alive and running with an isActive() call or an error may occur later in production when TypeCzech is turned off. 
if (type_czech.isActive()) {
  check_error= "check-error"
  error_loc = "error-location-TEST-F-104"
  var_value = "variable-value"
  expected_mess = "expected-message"
  type_czech.check_assert(check_error, error_loc, var_value, expected_mess)
}
Note that check_assert() usually works outside of the general linked up PRE_check_functions(). This function is needed to follow the chosen error delivery method for TypeCzech of 'THROW-EXCEPTIONS', 'LOG-ERRORS', or 'NO-CHECKING'.

Note that if checking is turned off, via TypeCzech.disableTests(), or with TypeCzech('NO-CHECKING') then all checking functions below will return immediately without doing any work.
  • checkParam_type()
  • checkParam_typeEither()
  • checkParam_typeExtra()
  • checkArgs_typeEach()
  • checkParam_empty()
  • checkParam_emptyEither()
  • checkParam_emptyExtra()
  • checkArgs_emptyEach()
  • check_interface()
  • check_mutatedSnapshot()
  • check_buildSnapshot()
  • check_assert()
